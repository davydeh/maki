# maki v0.2 Workspace Flow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add project opening, recent-project restore, multi-window routing, and the in-app config wizard while keeping the existing dark terminal workspace UI intact.

**Architecture:** Rust owns native services for app-state persistence, project inspection, config generation, file saving, and project-window routing keyed by canonical folder path. React owns the visible session state machine and switches between picker, wizard, and workspace views while reusing the existing terminal components once a config is available.

**Tech Stack:** Tauri v2, Rust, React 19, TypeScript, Vite, portable-pty, serde/serde_yaml, native Tauri dialogs/windows, Vitest + React Testing Library, Rust unit tests
