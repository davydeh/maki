## What Was Done This Session

### Working Features
- **Terminal**: PTY output pipeline fixed (64KB buffer, UTF-8 boundary preservation, string payload). Claude Code renders correctly with Shift+Enter support.
- **Tabs**: Two-tier layout — shell tabs (top, rounded pill, Ghostty-inspired) + command bar (bottom). Shell tabs show project name, double-click to rename, Cmd+1-9 to switch.
- **Command palette**: Cmd+P opens a search modal for one-off commands. Arrow keys + Enter to run.
- **Command bar**: Shows autostart commands as pills with status dots. Transient (one-off) commands appear temporarily, auto-removed 10s after exit.
- **Design system**: Flat catppuccin mocha, no gradients/shadows. Antialiased fonts.
- **Shell keybindings**: Cmd+K (clear), Cmd+Left/Right (line), Option+Left/Right (word), Option+Backspace (delete word).
- **Wizard**: Compact one-row-per-command, rounded checkboxes, minimal sections.
- **Native menu**: File menu with New Window/Tab, Split Right/Down, Close.
- **Distribution**: `npm run tauri build` produces .app + .dmg. GitHub Actions workflow on `v*` tags.

### Known Bugs (to fix next session)

1. **Starship prompt arrow broken (again)**: The pointed arrow segment (U+E0B0 powerline glyph) renders as a rectangle. This happened when WebGL was disabled for perf. WebGL is re-enabled now but the issue persists. Root cause: the DOM renderer doesn't handle powerline glyphs. Fix: ensure WebGL addon loads for shell tabs (it's in the code but may be failing silently in WKWebView).

2. **Command pills too narrow**: The pills were simplified too much. They need wider min-width and better proportions.

3. **Clicking stopped command pill does nothing**: `handleRunCommand` only focuses running commands. Stopped/errored pills should also focus (show their last terminal output) AND offer a restart action.

4. **Stop button UX is bad**: The small red ■ on hover is hard to use. Need proper sized play/stop buttons. Consider a split button (click to focus, secondary action to stop/start) or always-visible icon buttons.

5. **Performance**: WebGL disabled globally due to GPU context limit with many tabs. Need to selectively enable WebGL only for the active terminal, or use canvas addon as fallback.

### Architecture Notes

- `src/App.tsx` — Main layout: TabBar → Terminal area → CommandBar → StatusBar. Owns tab state, keyboard shortcuts, menu event listener.
- `src/components/TabBar.tsx` — Exports `TabBar` (shell tabs), `CommandBar` (autostart pills), `CommandLauncher` (Cmd+P modal).
- `src/components/TerminalView.tsx` — xterm.js wrapper. PTY spawn in init effect (after listeners). Custom key handler for Shift+Enter CSI u, Cmd+K clear, etc.
- `src-tauri/src/pty.rs` — PTY reader with UTF-8 boundary preservation across reads. 64KB buffer.
- `src-tauri/src/lib.rs` — Tauri setup with native File menu.
- `src/themes.ts` — Theme has `inactiveTabBg` token added for tab bar.
- `src/types.ts` — Tab has `transient?: boolean` field for one-off commands.

### Key Design Decisions

- **Shells = browser tabs, Commands = bookmarks bar**: Interactive shells are the primary workspace (top). Commands are utilities (bottom). This scales to projects with 10+ commands.
- **Autostart vs one-off**: Autostart commands always show in the bar. One-off commands accessed via Cmd+P palette and appear transiently.
- **Single window for v0.2**: Multi-window routing exists in Rust but isn't exercised. Open Folder replaces the current project.
